````text
Krypt manages your environment variables across teams with end-to-end encryption. This guide gets you from zero to your first secret pushed in 5 minutes.

## Prerequisites

- A free [Krypt account](https://krypthq.com/signup)
- Node.js 20 or newer ([install Node](https://nodejs.org/))

## 1. Install the CLI

```bash
npm install -g @kryptorg/cli
```

If you get a permission error, prefix with `sudo`:

```bash
sudo npm install -g @kryptorg/cli
```

Verify the install:

```bash
krypt --version
```

## 2. Generate an API key

1. Go to [krypthq.com/dashboard/api-keys](https://krypthq.com/dashboard/api-keys)
2. Click **Generate Key**
3. Name it (e.g. "my-laptop")
4. Copy the key — it starts with `krypt_live_` and is shown only once

## 3. Log in from your terminal

```bash
krypt login --key krypt_live_<paste-your-key>
```

You should see:
````

✓ API key verified Config saved to ~/.krypt/config.json

````text

## 4. Create a project

In your Krypt dashboard:

1. Click **New Project**
2. Name it `my-app`
3. Save

## 5. Link your local folder to the project

In your project directory:

```bash
cd my-app
krypt init my-app
```

This creates a `.krypt` config file linking the folder to your project.

## 6. Push your first secrets

If you have an existing `.env` file, push it:

```bash
krypt push
```

Otherwise, create one:

```bash
echo "DATABASE_URL=postgres://localhost:5432/mydb" > .env
echo "API_TOKEN=secret-value" >> .env
krypt push
```

You'll see:
````

✓ Pushed 2 secret(s) to "my-app" (development)

````text

Your secrets are now encrypted and stored in Krypt.

## 7. Pull secrets back

On another machine, or in CI:

```bash
krypt pull
```

Krypt fetches your secrets and writes them to `.env`.

## 8. Run a command with secrets injected

```bash
krypt run -- node server.js
```

Krypt injects your secrets as environment variables and runs your command. No `.env` file required.
````
